QC Player and Codec Preview Requirements Specification (Stability Release)

Document ID: RP-QC-REQ-2026-03-01
Date: 2026-03-01
Status: Active

1. Scope

This specification defines required behavior for the desktop Listen and QC experience:

- Library ingest and root lifecycle
- Playlist and queue interaction
- Shared playback transport
- Codec preview (A/B/X)
- Batch export for codec artifacts
- Search, sorting, grouping, and keyboard shortcuts

Out of scope for this document:

- Publish workspace execution workflow changes
- External publisher connector rollout

2. Product Intent

The product must deliver a trustworthy post-mastering QC and playback workflow where:

- What the user hears in codec preview is the real selected codec path.
- Export status always reflects real encoder outcomes.
- Library and queue behavior remains stable under large catalogs and repeated interaction.
- The system is testable and maintainable as a long-lived desktop application.

3. Functional Requirements

FR-001: Real Codec Preview Truth Layer

- The preview session must support variants: bypass, codec_a, codec_b, blind_x.
- Variant selection must switch actual playback source, not only UI/session metadata.
- If the selected profile cannot be rendered, the command must return an explicit error.

Pass:
- Active variant changes both state and playback source path.

Fail:
- Active variant changes while playback source remains unchanged.

FR-002: Blind-X Determinism

- Blind-X must map to A or B deterministically per preview session.
- Reveal must return the exact mapped identity and preserve auditable state.

Pass:
- Repeated reads before reveal keep mapping hidden and stable.

Fail:
- Blind-X mapping changes unpredictably during one session.

FR-003: Batch Export Outcome Integrity

- Per-profile export statuses must be explicit and truthful.
- The system must not report completed when encoding did not happen.
- Required status set:
  - completed
  - failed_encoder_unavailable
  - failed_encode
  - failed_io

Pass:
- Missing encoder results in failed_encoder_unavailable.

Fail:
- Missing encoder results in completed output.

FR-004: QC Metric Integrity

- Integrated loudness must follow BS.1770 semantics.
- True peak (dBTP) must be computed through a true-peak path, not waveform max proxy.
- Stored track metrics must be finite and range-validated.

Pass:
- True peak is independently computed and persisted.

Fail:
- true_peak_dbfs is derived only from peak bins or sample max approximation.

FR-005: Library Root Lifecycle Correctness

- Dropped folder ingest must add root, scan, and autoplay first discovered track.
- Removing a root must remove tracks sourced from that root and prune stale queue/favorites/selection references.
- Reset library data must clear persisted local catalog and UI-derived state safely.

Pass:
- Track counts and queue/favorites stay consistent after root removal.

Fail:
- Removed root tracks remain visible or keep stale queue references.

FR-006: Queue and Playlist Control

- Queue mode and library mode toggling must work with native and fallback transport paths.
- Queue reorder must work via drag-drop and keyboard shortcuts.
- Batch actions must preserve deterministic insertion order.

Pass:
- Reorder and mode toggle work without false error banners.

Fail:
- Toggle path emits non-actionable playback errors when fallback mode is healthy.

FR-007: Shortcut Configuration

- Shortcuts must be editable in Settings and persisted.
- Conflicting bindings must be detected and visibly flagged.
- Focus search shortcut must move focus to track search input.

Pass:
- Rebound shortcut is effective after remount.

Fail:
- Binding changes are accepted but do not execute action.

FR-008: Search and Browse Scalability

- Search must support title, artist, album, and file_path relevance.
- Paging must preserve stable ordering across load-more operations.
- Search must avoid first-page ranking bias for large libraries.

Pass:
- Results stay stable across pagination and repeated query.

Fail:
- Later pages reorder or invalidate prior ranked results unexpectedly.

4. Non-Functional Requirements

NFR-001: Stability

- No infinite render/update loops in UI effects.
- Async polling and completion handlers must be idempotent.
- State transitions must avoid redundant setState cascades.

NFR-002: Maintainability

- Large orchestration logic must be split into focused controllers/hooks.
- Command contracts must remain typed and version-safe.
- Shared constants and feature flags must be centralized.

NFR-003: Testability

- Every new command behavior must include contract tests.
- Every fixed regression must include a dedicated test case.
- Deterministic test fixtures are required for ingest/preview/export paths.

NFR-004: Performance

- Search and ingest operations must remain responsive for large local libraries.
- Export and scan operations must run off UI-critical threads.
- Playback controls must not block on disk operations.

NFR-005: Security and Boundary Safety

- Frontend input is untrusted; backend validates all command inputs.
- Error payloads must remain sanitized for UI display.
- Path handling must remain canonicalized and bounded.

5. Quality Gates (Pass/Fail)

Gate A: Build and Test Gate

Pass:
- npm run test --workspace apps/desktop -- --run
- cargo test -p release-publisher-desktop --lib

Fail:
- Any red test in modified scope.

Gate B: Functional Truth Gate

Pass:
- Variant selection changes playback truthfully.
- Export outcomes are semantically accurate.
- Root lifecycle leaves no stale visible state.

Fail:
- UI state claims behavior not backed by backend outcome.

Gate C: Regression Safety Gate

Pass:
- No maximum update depth warnings in tests.
- Queue toggle fallback and drop-autoplay tests remain green.

Fail:
- Previously fixed loop/fallback defects reappear.
