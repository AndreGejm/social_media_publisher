# Triage Tickets

Store triage tickets in this directory:

- `docs/reports/triage/`

Use one file per bug report.

## File naming

Use:

- `YYYY-MM-DD-short-kebab-title.md`

Example:

- `2026-03-12-crash-on-export-open.md`

## Required format

Each ticket must use this exact section structure:

1. `TRIAGE NOTE`
2. `Title`
3. `Bug Report Summary`
4. `Observed Behavior`
5. `Expected Behavior`
6. `Severity`
7. `Bug Type`
8. `Surface Area`
9. `Initial Root-Cause Hypothesis`
10. `Likely Investigation Start Points`
11. `Module`
12. `Directory`
13. `Candidate Files`
14. `Candidate Functions`
15. `Evidence Used`
16. `Missing Information`
17. `Triage Confidence`
18. `Recommended Next Step`

Copy `TEMPLATE.md` in this folder for new tickets.
