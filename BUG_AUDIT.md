## Pass 3 Resolution Notes

- `Shared player bar disappears on About workspace` is fixed by keeping the shared player mounted throughout Release Preview mode, including `About`.
- `Settings clear-banner controls stay enabled when nothing can be cleared` is fixed by disabling `Clear Notice` and `Clear Error Banner` when their corresponding banner state is absent.
- `Browser preview raises an unexpected startup error banner when Tauri is unavailable` is fixed by normalizing `TAURI_UNAVAILABLE` handling in the timeout wrappers instead of surfacing raw invoke failures.
- `Hidden Video Workspace reacts to Library dropped media while its panel is not active` is fixed by subscribing the Video Workspace to native drop events only while that workspace is active and visible.
- `Drag-drop subscription resolves inconsistently in shell regression coverage` is fixed by making `subscribeToFileDropEvents` use a stable `@tauri-apps/api/webview` import that Vitest can mock consistently, while preserving browser/runtime fallback behavior.
- `Cold restart preserves stale queue, favorite, and publish-selection ids` is fixed by pruning invalid persisted track references once during initial shell restore, instead of waiting for a later root-removal action.
- `Browser-fallback `Play Now` loads a track without entering an active playing state` is fixed by setting `playerIsPlaying` as soon as fallback autoplay succeeds, so the shared transport reports active playback immediately.
- `Passive Video Workspace navigation surfaces warnings before any render action` is fixed by suppressing idle browser-preview diagnostics errors and only surfacing the empty output-directory alert after preflight/failure states or real output input.
- `Packaged-runtime synthetic drag/drop does not surface the dropped file in catalog state reliably` remains deferred as future work because the listener bridge now works, but the remaining import/catalog convergence issue is not high enough value to risk broader state regressions in this pass.
