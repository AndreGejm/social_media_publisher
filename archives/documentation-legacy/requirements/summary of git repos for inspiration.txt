Summary of Open-Source Repositories for Multi‑Codec QC Player

The following open‑source projects provide building blocks for a real‑time QC player that can preview different streaming codecs and export multiple formats. Each repository solves a specific part of the pipeline (loudness/true‑peak measurement, FFT computation, codec processing, BPM detection, lock‑free buffering) and is worth reviewing before implementing your own version. Citations from each project’s README are provided as evidence.

Loudness / True‑Peak Measurement
Repository	Highlights	Key Evidence
jiixyj/libebur128 (ANSI‑C)	Implements the EBU R128 broadcast loudness standard and loudness range measurement; provides Momentary, Short‑term, Integrated LUFS readings and True Peak scanning; portable C code with MIT licence	README notes that the library implements M/S/I modes, loudness range, true‑peak scanning and is portable ANSI‑C
sdroege/ebur128 (Rust port)	Rust crate compatible with the C version; passes all tests defined in EBU Tech 3341/3342; offers the same APIs as libebur128 and even exposes a C API	The README states that it is a Rust port of the C implementation, providing momentary, short‑term, integrated loudness and loudness range, true peak scanning and supporting all sample rates

xiph/vorbis	Reference implementation of the Ogg Vorbis codec; the Vorbis specification is in the public domain and free of royalties; the package contains libvorbis (codec library), libvorbisfile (convenience library), libvorbisenc (encoding API), and example programs	README states that Vorbis is a general‑purpose audio format belonging to the public domain; the package includes libraries for encoding and decoding plus example code

Additional Considerations

FFmpeg/libav – Although not included above due to licensing complexity, FFmpeg provides a wide variety of codecs (AAC, MP3, etc.). If licensing is acceptable, linking to FFmpeg’s libraries (libavcodec, libavformat) could give you access to additional streaming codec profiles.

cpal – Cross‑platform audio I/O library in Rust. Useful if you build your own playback engine rather than using JUCE.

These projects demonstrate architectures and patterns relevant to your QC player:

libebur128/ebur128 show how to implement the ITU‑R BS.1770 loudness algorithm and true‑peak scanning.

RustFFT demonstrates a pure‑Rust, highly optimised FFT suitable for real‑time spectra.

xiph/opus and xiph/vorbis illustrate how to encode/decode audio for preview; their APIs can serve as a guide for integrating streaming codecs with delay compensation.

aubio provides algorithms for BPM/tempo detection, onset detection and other music analysis tasks.

direct_ring_buffer reveals how to design a lock‑free, single‑producer/single‑consumer buffer for bridging DSP and GUI threads without blocking or allocations.
